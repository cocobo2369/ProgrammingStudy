# 모놀리식 아키텍처 vs 마이크로서비스아키텍처

## 모놀리식 아키텍처

> 특징
<pre>
1. 하나의 app안에 모든 컴포넌트를 포함하는 구조
2. 개발과 배포가 쉽다.
</pre>

> 한계
 <pre>
1. 비효율적인 분산 전략이다.
2. 소스 코드가 거대해서 전체를 이해해야 수정이 가능
3. 새로운 기술 적용이 어렵다.
</pre>

> 요구 사항
<pre>
1. 코드 구조가 독립적이어야 한다.
2. 기능별 분산 구조여야 한다.
3. 기능별 최적화된 기술 적용이 가능해야 한다.
</pre>



## 마이크로 서비스

> 특징
<pre>
1. 개발과 배포에 상호 독립적이다.
2. 독립적인 데이터 저장소를 가질 수 있다.
3. 각자 가진 다른 네트워크 기술로 통신할 수 있다.
</pre>

> 단점
<pre>
1. 공유 자원 접근이 어렵다.
2. 배포와 실행이 복잡하다.
3. 분산 시스템을 구현하기 어렵다(메세징큐)
</pre>

> 마이크로서비스와 node.js
<pre>
1. node.js에는 메세지-큐 기능이 자체적으로 들어있다.
2. node.js에는 경량화된 고성능 서버 개발에 최적화된 프레임워크다.
</pre>
